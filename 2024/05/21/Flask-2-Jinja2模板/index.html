转义字符过滤器

在 HTML 中 <、>、空格、\t 等都有特殊的含义或者特殊的显示效果。

根据直觉，HTML 中会将 info 字符串的数据原样输入，而不会将其当成 js 代码执行

，事实确实也是如此：

如果要让 info 中的字符串被当作 js 代码来执行，应该使用 safe 过滤器

为了更精细的控制转义与不转义，可以使用 autoescape 和 escape 配合 来控制

autoescape 设定了“大环境”，autoescape false 代表大环境是不转义，autoescape Trueue 代表大环境是转义，而 escape 代表无论“大环境”是什么，都强制转义。

此时第一行的 info 会被当成 js 代码执行，也就是“大环境”不转义；

​ 第二行的 info 会被当成字符串显示，也就是“大环境”不转义，但 escape 强制转义；

​ 第三行的 info 会被当成字符串显示，也就是“大环境”转义了；
